# Memory Rules

## 1. Long-term memory
适合保存：
- 用户长期偏好
- 平台开发原则
- 常用输出格式
- 固定项目路径

## 2. Working memory
适合保存：
- 当前阶段目标
- 当前服务状态
- 当前正在推进的 batch
- 最近一轮关键结果

## 3. Do not store
不要长期保存：
- 临时日志
- 一次性错误堆栈
- 短时环境细节
- 无复用价值的随机输出

## 4. Agent-facing memory discipline
- 先总结,再存储
- 只存高价值上下文
- 记录"完成了什么 / 还缺什么 / 下一步是什么"
